DOM Tree

Accessibility Tree

Accessibility Tree

  1. Дерево доступности строится для интерактивных элементов
  2. Семантика имеет значение: атрибуты role, name, value, alt
  3. Семантика реально имеет значение: headings, <header>,<main>, <label>
  4. Я серьезно: <form>
  5. Семантика решает: <a>

А зачем тогда ARIA-атрибуты?

Сложные компоненты

  1. Календари
  2. Date / Time / DateTime pickers
  3. Комбобоксы
  4. Очень кастомные нативные элементы

ARIA-атрибуты отвечают за семантику в тех местах, где нативные средства HTML5 бессильны

В остальных случаях они вам не нужны

Кнопка с иконкой

      <button type="button" class="filter"
              data-dismiss="alert" aria-label="Toggle filter">
          <span aria-hidden="true"></span>
      </button>

span не попадет в дерево доступности

Дерево

      <ul role="tree" aria-expanded="false"
              aria-label="Expand tree node">
          <li role="treeitem" aria-label="Tree node"></li>
          <li role="treeitem" aria-label="Tree node"></li>
      </button>

Бооольше атрибутов

  1. aria-labelledby - ссылка на описание в другой ноде
  2. aria-live, aria-atomic, aria-relevant - настройка оповещения об изменениях в ноде. Например, добавление элемента в список
  3. aria-disabled - блокировка элемента
  4. aria-invalid - валидация элемента
  5. ... и множество других, характерных для той или иной роли
